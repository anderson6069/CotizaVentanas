Ingreso de Datos:

El sistema debe permitir ingresar el estilo de la ventana (O, XO, OXXO).
El sistema debe permitir ingresar las dimensiones de la ventana (ancho y alto).
El sistema debe permitir seleccionar el tipo de acabado de aluminio (Pulido, Lacado Brillante, Lacado Mate, Anodizado).
El sistema debe permitir seleccionar el tipo de vidrio (Transparente, Bronce, Azul) y si el vidrio será esmerilado.

Cálculo de Costos:

El sistema debe calcular el costo del aluminio basándose en las dimensiones de la ventana y el acabado seleccionado.
El sistema debe calcular el costo del vidrio, considerando el tipo de vidrio y el esmerilado si se aplica.
El sistema debe calcular el costo de chapas y esquinas según el estilo de ventana.
El sistema debe calcular el costo total de la ventana sumando todos los costos (aluminio, vidrio, chapas, esquinas).
El sistema debe aplicar un descuento del 10% si el pedido es superior a 100 ventanas.

Visualización de Resultados:

El sistema debe mostrar un desglose de los costos calculados.
El sistema debe mostrar el costo total después de aplicar el descuento (si aplica).

Interfaz de Usuario:

El sistema debe proporcionar una interfaz de texto para ingresar datos y mostrar resultados.

Usabilidad:

El sistema debe ser fácil de usar, con una interfaz intuitiva para ingresar datos y visualizar resultados.
La interfaz debe manejar errores de entrada (por ejemplo, dimensiones negativas) y proporcionar mensajes claros.

Desempeño:

El sistema debe realizar los cálculos de manera eficiente y en tiempo real.
La aplicación debe manejar múltiples entradas de datos y cálculos sin demoras significativas.

Mantenibilidad:

El código debe estar bien documentado y seguir las buenas prácticas de programación (PEP-8).
El sistema debe estar estructurado de manera modular para facilitar su mantenimiento y futuras actualizaciones.

Portabilidad:

El sistema debe ser compatible con el entorno de desarrollo especificado (Python en Visual Studio Code).

Seguridad:

El sistema debe validar los datos de entrada para evitar errores de cálculo o resultados inesperados.
